Instructions

You are to make as many optimizations as possible to your Frogger game from COMP 20 last semester (or whatever HTML5 game you choose) using the techniques that we discussed in class. A few requirements:

  1. You must incorporate the HTML5 offline application cache for your game.
  2. Your game must run on both the latest versions of Google Chrome and Firefox.
  3. Your game (HTML + CSS + JS + images) cannot be greater than 500 KB.

Your game does not need to connect to a high score server (e.g., last year's Assignment 5). You should remove any references to it.

The point of this assignment is not to tinker a set of technologies but to apply a few important engineering concepts such as caching. I also need to stress that size and resources matter.

The README

The most important part of this assignment is the README. Your README must prove to me that the performance of your game has vastly improved compared to what you wrote or what was given to you. Specifically, you must answer the following questions:

  1. What optimization techniques did you use?
  2. How did you ensure that the HTML5 offline application cache worked?
  3. What tools did you use to test the performance of this updated version of your game? Using no tools for this assignment is not acceptable.
  4. What performance aspects have been improved (e.g., loading time, size)? Please provide numbers, percentages, or letter grades.
  5. Are there potential performance issues?

Submitting the Assignment

Push all the files to the private repository that I created for you in a folder named performance. Say that your private repository is named mchow-comp120, make sure all the files are pushed to mchow-comp120/performance.
